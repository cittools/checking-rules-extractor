A string literal shall not be modified [JSF-151.1-2]

DESCRIPTION 

A string literal shall not be modified. Rule reports violations if
a string literal is assigned to non-const variable, parameter or expression.
SINCE 

v7.1
BENEFITS 

The effect of attempting to modify a string literal is undefined.
EXAMPLE 

void moo(char * p)
{
}

void foo()
{
	char* c1 = "Hello";    // Violation
	char c2[] = "Hello";   // Violation
	char c3[6] = "Hello";  // Violation
	char* c12;
	c12 = "Hello";         // Violation
	moo("Hello");          // Violation
}
REPAIR 

void moo(const char * p)
{
}

void foo()
{
	const char* c1 = "Hello";    // OK
	const char c2[] = "Hello";   // OK
	const char c3[6] = "Hello";  // OK
	const char* c12;
	c12 = "Hello";               // OK
	moo("Hello");                // OK
}
