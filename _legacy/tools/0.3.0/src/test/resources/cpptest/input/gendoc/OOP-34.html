Check for assignment to self in operator= [OOP-34-4]

DESCRIPTION 

This rule checks your code for aliasing in assignment operators. Not checking
for assignment to self in operator= can free resources that might be needed
during the process of allocating new resources.
See also: CODSTA-CPP-02, CODSTA-CPP-21, CODSTA-CPP-24, MRM-04, MRM-37, MRM-40,
          MRM-47, OOP-27, OOP-30
BENEFITS 

Checking for assignment to self may also save you a lot of work that you
would otherwise have to do to implement assignments.
EXAMPLE 

class A  {
public:
    A( ) { }
    A& operator=( A& a ) {  // Violation
        return *this;
    }
};
REPAIR 

class A  {
public:
    A( ) { }
    A& operator=( A& a ) {  // OK
        if (&a != this) {
            // ...
        }
        return *this;
    }
};
