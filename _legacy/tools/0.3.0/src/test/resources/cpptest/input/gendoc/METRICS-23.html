Avoid deep nesting [METRICS-23-3]

DESCRIPTION 

"Excessively long functions and nested code blocks are often 
caused by failing to give one function one cohesive responsibility,
and both are usually solved by better refactoring."
BENEFITS 

Rule improves easier code understanding, and maintain, and often needlessly so.
EXAMPLE 

class A {
    void foo( int x ) {
        if (x > 0) {                    // OK
            if (x > 1) {                // OK
                if (x > 2) {            // OK
                    if (x > 3) {        // OK
                        if (x > 4) {    // OK
                            if (x > 5) { // Violation
                            }
                        }
                    }
                }
            }
        }
    }
};
REPAIR 

Do not use excessive block nesting depth (5 levels)
REFERENCES 

Herb Sutter, Andrei Alexandrescu, "C++ Coding Standards," Addison-Wesley,
