Identifiers shall be given for all of the parameters in a function prototype declaration [MISRA2004-16_3-3]

DESCRIPTION 

Names shall be given for all the parameters in the function declaration 
for reasons of compatibility, clarity and maintainability.
BENEFITS 

Rule improves readability and clarity of code.
EXAMPLE 

void foo1( void );          /* OK */
void foo2( int a, int );    /* Violation */
void foo3( int a, int ){}   /* OK */
REPAIR 

void foo1( void );          /* OK */
void foo2( int a, int b );  /* OK */
void foo3( int a, int ){}   /* OK */
REFERENCES 

MISRA-C:2004 Guidelines for the use of the C language in critical systems
Chapter 6, Section 16
