switch statement which results are invariant shall not be permitted [OPT-21-3]

DESCRIPTION 

Avoid writing useless switch statements.
See also: MISRA2004-15_5, OPT-22
SINCE 

v7.1
BENEFITS 

If a switch statement yields a result that can be proven to be always the same, 
it is highly likely that there is a programming error.
EXAMPLE 

int foo(int i) {
    switch(i) {    /* Violation */
        case 1:
        case 2:
        default:
            return 0;
    }
}
REPAIR 

int foo(int i) {
    return 0;      /* OK */
}
