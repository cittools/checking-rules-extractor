The type char shall always be declared as unsigned char or signed char [PORT-09-3]

DESCRIPTION 

"Do not assume that a char is signed or unsigned."
BENEFITS 

Complying with this rule leads to safer and more readable code.
EXAMPLE 

void foo()
{
    char a; // Violation
};

char goo();     // Violation
REPAIR 

void foo()
{
    signed char a;   // OK
};

unsigned char goo();     // OK

Ellemtel Coding Standards
