Avoid expressions in return statements of void functions [MISRA-084-3]

DESCRIPTION 

For functions with void return type, 
return statements shall not have an expression.
BENEFITS 

Rule prevents from undefined behaviour (the compiler may not give an error).
EXAMPLE 

/* compilable by 'c' compiler */
void foo2() {
  return 1;  /* Violation */
}

void foo3() {
  int a=0;
  return a;  /* Violation */
}

void foo4(int a) {
  return a;  /* Violation */
}
REPAIR 

/* compilable by 'c' compiler */

void foo1() {
  return;    /* OK */
}
