A project shall not contain unused variables [MISRA2008-0_1_3_a-3]

DESCRIPTION 

This rule checks if all declared local variables are used.
See also: OPT-06
NOTES 

Objects can perform some actions in their constructors. Even if they are not
used later they might have done their job during creation. This rule does not 
consider such objects as "used" and will report a violation.
If it is not desired to report a violation messages on local objects 
rule "OPT-06" should be used instead of this rule.
BENEFITS 

Eliminating unused local variables increases efficiency and legibility.
DRAWBACKS 

Rule can also report false positive when a variable is used only within asm block
and a file is compiled with gcc.
EXAMPLE 

class A {
public:
    void displBalance();
};

void foo( ) {
    A a;        // Violation 
}

void func( ) {
    int i = 0;  // Violation
}
REPAIR 

class A {
public:
    void displBalance( );
};

void foo( ) {
    A a;
    a.displBalance( );  // OK
}

void func( ) {
    int i = 0;          // OK
    i++;
}
REFERENCES 

1. MISRA C++:2008 - Guidelines for the use of the C++ language in critical
   systems, Chapter 6, Section 0, Rule 0-1-3
