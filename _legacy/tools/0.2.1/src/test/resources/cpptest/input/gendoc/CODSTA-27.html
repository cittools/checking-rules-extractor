Avoid functions that modify global variables [CODSTA-27-3]

DESCRIPTION 

This rule is purely informational and will report a message
when function modifies global variables.
BENEFITS 

In multithread environment using global
variables makes code extremely error prone.
EXAMPLE 

int theGlob;
void foo( int iVar )
{
    theGlob = -1;       // Violation
    if (iVar == 0 )
	{
        theGlob = 0;    // Violation
    } else {
        theGlob = 1;    // Violation 
    }
    return;
}
REPAIR 

void foo( int iVar )
{
   static int theLocal;
    theLocal = -1;       // OK
    if (iVar == 0 )
	{
        theLocal = 0;    // OK
    } else {
        theLocal = 1;    // OK
    }
    return;
}
