A null statement shall only occur on a line by itself, and shall not have any other text on the same line [MISRA-054-3]

DESCRIPTION 

"Null statements should not normally be deliberately included, but where they 
are used they shall appear on a line by themselves and not with any other
text (including comments). Following this rule enables a static checking tool
to warn of null statements appearing on a line with other text, which would 
normally indicate a programming error."
BENEFITS 

Rule provides readability and clarity of code.
EXAMPLE 

void foo1()
{
    ;        // Violation
    int a;; // Violation
    /* Violation */ ;
}
REPAIR 

void foo1()
{
    ;        // OK
    int c
    ;        // OK
    for(;;); // OK
}
