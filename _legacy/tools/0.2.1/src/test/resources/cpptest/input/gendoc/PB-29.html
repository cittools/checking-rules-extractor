The left-hand operand of a right-shift operator shall not have a negative value [PB-29-2]

DESCRIPTION 

Rule reports a violation message if left-hand operand of right-shift operator 
is signed type, negative constant, or is preceded by "-" operator (-a).
SINCE 

v7.1
BENEFITS 

For e1 >> e2, if e1 has a signed type and a negative value, the value
of (e1 >> e2) is implementation-defined.
EXAMPLE 

void foo(signed int param)
{
    int i = 1, j = 1, k = 1;
    signed int m = -1;

    k = param >> j;          // Violation
    k = i >> j;              // Violation
    k = -i >> j;             // Violation
    k = -1 >> j;             // Violation
    k = m >> j;              // Violation
    m>>=j;                   // Violation
}
REPAIR 

void zoo(unsigned int param)
{
    int i = 1, j = 1, k = 1;
    unsigned int m = 1;
    
    k = param >> j;         // OK
    k = 1 >> j;             // OK
    k = m >> j;             // OK
    m>>=j;                  // OK
}
REFERENCES 

JOINT STRIKE FIGHTER, AIR VEHICLE, C++ CODING STANDARDS
Chapter 4.21 Operators, AV Rule 164.1
