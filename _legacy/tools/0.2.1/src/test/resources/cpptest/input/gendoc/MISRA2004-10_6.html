A "U" suffix shall be applied to all constants of unsigned type [MISRA2004-10_6-5]

DESCRIPTION 

"Signedness of constants should be explicit. Consistent signedness
is an important principle in constructing well formed expressions.
If a constant is of an unsigned type, it is helpful to avoid
ambiguity by applying a “U” suffix."
The rule reports a violation message if to an unsigned variable is assigned
a constant without 'U' suffix.
SINCE 

v7.0
BENEFITS 

Rule improves readability and maintainability of code.
EXAMPLE 

void foo()
{
    unsigned int a = 10;       /* Violation */
    unsigned int b;
    b = 10;                    /* Violation */
}
REPAIR 

void foo()
{
    unsigned int a = 10U;       /* OK */
    unsigned int b;
    b = 10U;                    /* OK */
}
REFERENCES 

MISRA-C:2004 Guidelines for the use of the C language in critical systems
