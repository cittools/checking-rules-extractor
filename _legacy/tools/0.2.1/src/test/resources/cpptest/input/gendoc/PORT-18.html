Don't define i64 or L suffixes directly [PORT-18-3]

DESCRIPTION 

This rule checks whether you use i64 or L suffixes directly. Do not use i64 or
L suffixes directly because it generally does not work well. Try using Macros
for compatibility.
See also: PORT-18
BENEFITS 

Using i64 or L suffixes directly generally does not work well.
Try using Macros for compatibility.
EXAMPLE 

#define V1 0x000000000000000cL          // Violation
REPAIR 

#ifdef _WIN64
   #define CONST3264(a) (a##i64)
 #else
   #define CONST3264(a) (a##L)
#endif

#define V1 CONST3264(0x000000000000000c) // OK
