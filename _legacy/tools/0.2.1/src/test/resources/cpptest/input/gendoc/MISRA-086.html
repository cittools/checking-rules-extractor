If a function return error information, then that error information should be tested [MISRA-086-5]

DESCRIPTION 

If a function return error information, 
then that error information should be tested.

This rule assumes that error information is returned as function
return value, any time return type is integer, 
and it triggers when this return value is not somehow used.
BENEFITS 

Rule prevents errornous code and improves quality.
EXAMPLE 

int foo1();
void foo2();
int foo3();

void moo() {
    int i;
     
    foo1();           /* violation */  
    if (1) {
	foo1();       /* violation */
    }
    
    if (foo1()) {
	foo3();       /* violation */
    }
}
REPAIR 

int foo1();
void foo2();
int foo3();

void moo() {
    int i;
    i=foo1();         /* OK */
    
    (void)foo1();     /* OK */  
    foo2();           /* OK */
    if (foo1()!=0) {} /* OK */    
}
