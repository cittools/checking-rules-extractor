A project shall not contain unused variables [MISRA2008-0_1_3_b-3]

DESCRIPTION 

This rule checks whether any of the declared local variables is not used
or has no side effect.
See also: OPT-02
BENEFITS 

Eliminating unused local variables increases efficiency and legibility.
DRAWBACKS 

Rule can report false positive when a variable is used only within asm block
and a file is compiled with gcc.
EXAMPLE 

void func() {
    int i = 0;    // Violation
}

int i = 0;
class A {
public:
    A(){};
};

class B {
public:
    B(){i++;};
};

void goo() {
    A a;	// Violation
    B b;	// OK
}
REPAIR 

void func() { // OK
}

int i = 0;
class A {
public:
    A(){};
};

class B {
public:
    B(){i++;};
};

void goo() {  // OK
    B b;
}
REFERENCES 

1. MISRA C++:2008 - Guidelines for the use of the C++ language in critical
   systems, Chapter 6, Section 0, Rule 0-1-3
