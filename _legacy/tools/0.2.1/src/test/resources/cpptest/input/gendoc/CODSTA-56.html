All 'case' and 'default' labels of switch statement should have an explicit break or a return statement, or 'fall through' comment [CODSTA-56-3]

DESCRIPTION 

All 'case' and 'default' labels of switch statement should have
an explicit break or a return statement, or 'fall through' comment
containing "fall through" phrase (checked case insensitive).


Exceptions:

Rule does not report a violation message for empty labels.
SINCE 

v7.2
BENEFITS 

Rule prevents unpredictable program behaviour.
EXAMPLE 

void MyFunction(int i) 
{
    switch(i) 
    {
    case 1:           // Violation
        i++;
    case 2:
    case 3:           // Violation
        i++;
        /* wrong comment */

    default:          // Violation
        ;
    }
}
REPAIR 

void MyFunction(int i) 
{
    switch(i) 
    {
    case 1:            // OK
        i++;
        break;             
    case 2:            // OK - empty label
    case 3:            // OK
        i++;
        /* fall through */
    default:           // OK
        break;
    }
}
