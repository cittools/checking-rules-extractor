A cast should not convert a pointer type to an integral type [MISRA2008-5_2_9-4]

DESCRIPTION 

"The size of integer that is required when a pointer is converted to an integer 
is implementation-defined. Casting between a pointer and an integer type should 
be avoided where possible, but may be unavoidable when addressing memory mapped 
registers or other hardware specific features."


Exceptions:

The rule allows to cast to UINT_PTR or INT_PTR type. These types are integral
types that scale to the size of a pointer for both 32-bit and 64-bit Windows.
BENEFITS 

Prevents undefined or implementation-defined behaviour.
EXAMPLE 

void foo( ) {
    int* pi;
    int i;
    i = (int) pi;    // Violation
}
REPAIR 

Do not cast pointers to non-pointers.
