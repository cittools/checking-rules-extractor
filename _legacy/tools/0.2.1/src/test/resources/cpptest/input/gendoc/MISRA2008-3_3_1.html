Objects or functions with external linkage shall be declared in a header file [MISRA2008-3_3_1-3]

DESCRIPTION 

"Placing the declarations of objects and functions with external linkage
in a header file documents that they are intended to be accessible from other
translation units. If external linkage is not required, then the object or
function shall either be declared in an unnamed namespace or declared static."


Exceptions:

This rule does not apply to main, or to members of unnamed namespaces.
SINCE 

v7.2
BENEFITS 

Rule reduce the visibility of objects and functions.
EXAMPLE 

// file.cpp
int a1 = 0;   // Violation
void fun(){}  // Violation
REPAIR 

// file.h
extern int a1;      // OK
extern void fun();  // OK

// file.cpp
#include "file.h"
int a1 = 0;   // OK
void fun(){}  // OK
REFERENCES 

MISRA C++:2008 Guidelines for the use of the C++ language in critical systems
Chapter 6, Section 3, Rule 3-3-1
