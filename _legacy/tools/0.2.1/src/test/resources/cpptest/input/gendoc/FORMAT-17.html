There shall be no white space following a primary operator [FORMAT-17-3]

DESCRIPTION 

This rule checks whether there is no white space following a primary operator.
See also: FORMAT-18
BENEFITS 

This promotes continuity in the relationship between the operators
and the members/elements on which they act. These operators are used
to access members/elements and cannot be considered separately from them.
EXAMPLE 

struct MyStruct {
   int x;
};

void foo(struct MyStruct* myObj) {
 // ...
 myObj-> x; // Violation
}
REPAIR 

struct MyStruct {
   int x;
};

void foo(struct MyStruct* myObj) {
 // ...
 myObj->x; // OK
}
